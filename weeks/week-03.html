Week 3 of 8 작업 메모

Week 3

CLAUDE.md를 작업 메모처럼 활용하기

이미 배포된 CLAUDE.md를 실제 작업에서 참고하고, 부족한 내용은 부담 없이 개선 후보로 남기는 방법을 다룹니다.

3줄 요약

  • CLAUDE.md는 새로 만드는 문서가 아니라 이미 있는 작업 메모를 Claude가 참고하게 하는 장치입니다.
  • 작업 전에는 관련 내용을 확인하게 하고, 작업 후에는 빠뜨린 부분이 없는지 다시 봅니다.
  • 오래된 테스트 명령이나 애매한 규칙을 발견하면 개선 후보로 남깁니다.

이번 주 포인트

CLAUDE.md는 한 번 만들어두고 끝나는 문서가 아닙니다. Claude Code가 팀에서 자주 쓰는 약속을 참고하게 하고, 실제 작업에서 발견한 빈틈을 가볍게 되돌려주는 메모에 가깝습니다.

우리 팀은 이미 CLAUDE.md를 배포해서 사용하고 있습니다. 그래서 이번 글은 “어떻게 처음 만들까”보다 “작업할 때 어떻게 가볍게 참고할까”에 가깝습니다. 좋은 메모가 있어도 Claude에게 그 내용을 확인해달라고 요청하지 않으면 실제 작업에서는 빠질 수 있습니다.

핵심은 간단합니다. 작업을 시작할 때 CLAUDE.md 내용을 먼저 확인하게 하고, 작업이 끝나면 빠뜨린 내용이 없는지 다시 보는 것입니다. 그리고 문서와 실제 코드가 어긋나는 부분을 발견하면 개선 후보로 가볍게 남깁니다. 팀원이 같은 문제를 반복해서 만나지 않게 만드는 것이 목적입니다.

1. 작업 시작 전에 메모를 확인하게 하기

Claude Code는 대화 내용과 직접 읽은 파일을 바탕으로 판단합니다. 그래서 중요한 작업을 맡길 때는 관련 메모를 먼저 확인하게 하는 한 줄이 도움이 됩니다. 

바로 구현하지 말고 먼저 CLAUDE.md를 확인해줘.
이 작업에 참고하면 좋을 팀 규칙, 테스트 명령, 주의할 파일을 정리한 뒤
수정 계획을 제안해줘.

2. CLAUDE.md와 결과를 다시 비교하기

구현이 끝난 뒤에도 CLAUDE.md는 쓸 수 있습니다. Claude에게 변경 diff를 CLAUDE.md와 비교해 다시 보게 하면 리뷰 전에 놓친 부분을 줄일 수 있습니다. 

이번 diff를 CLAUDE.md와 비교해서 다시 리뷰해줘.
어긴 규칙, 빠진 테스트, 위험한 변경 범위가 있으면 먼저 알려줘.

3. 문서가 틀렸거나 부족할 때 남기는 피드백

실제 작업을 하다 보면 CLAUDE.md에 없는 규칙, 오래된 명령, 애매한 표현을 발견할 수 있습니다. 이때 개인이 대충 기억하고 넘어가면 다음 사람도 같은 문제를 만납니다.

그냥 넘기는 방식

테스트 명령이 틀린 것 같은데
이번에는 내가 알아서 돌릴게.

더 나은 방식

CLAUDE.md의 테스트 명령이 현재 Gradle 설정과 맞지 않아.
이번 작업에서는 ./gradlew test를 사용했고,
CLAUDE.md에는 이 명령으로 수정하는 것을 제안해줘.

문서 수정 권한이 없더라도 괜찮습니다. PR 설명이나 팀 채널에 “이번 작업에서 발견한 CLAUDE.md 개선 후보”로 남기는 것만으로도 다음 작업이 편해집니다.

4. 개인 취향과 팀 메모를 섞지 않기

CLAUDE.md는 팀원이 함께 참고하는 메모입니다. 개인이 선호하는 답변 방식이나 작업 스타일은 개인 메모리에 두고, 팀 문서에는 빌드, 테스트, 설계, 리뷰처럼 모두가 자주 참고할 내용을 남기는 편이 좋습니다.

5. 길어질수록 가끔 정리가 필요합니다

CLAUDE.md가 길어지면 모든 내용이 같은 중요도로 반영되기 어렵습니다. 오래된 규칙, 특정 사건의 회고, 예외가 많은 설명은 가끔 정리해두면 좋습니다. 목적은 문서를 길게 만드는 것이 아니라 Claude가 참고할 내용을 찾기 쉽게 유지하는 것입니다.

팀에서 가볍게 맞춰보면 좋은 것

작업 전에는 CLAUDE.md를 확인하게 하고, 작업 후에는 문서와 실제 코드가 어긋난 부분을 개선 후보로 남깁니다.

그대로 써봐도 되는 요청

작업을 시작할 때 관련 메모를 먼저 확인하게 하는 요청입니다.

바로 구현하지 말고 먼저 CLAUDE.md를 확인해줘.
이 작업에 참고하면 좋을 테스트 명령, 조심할 범위,
리뷰 포인트를 정리한 뒤 수정 계획을 제안해줘.

이번 주 실천

버그 수정 요청 앞에 “CLAUDE.md를 먼저 확인해줘”를 붙이고, 작업 후에는 문서 개선 후보가 있는지 같이 물어봅니다.

이번 주에 가볍게 남겨볼 것

실제 작업 하나를 골라 CLAUDE.md가 도움이 됐는지 확인해봅니다. 맞지 않는 테스트 명령, 빠진 리뷰 포인트, 애매한 표현이 있으면 “CLAUDE.md 개선 후보”로 PR 설명이나 팀 채널에 남깁니다.
