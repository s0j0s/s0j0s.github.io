Claude Code Engineering Week 1

8-week newsletter

Claude Code를 개인 도구에서 팀의 Engineering System으로

회사에서 이미 Claude Code를 쓰고 있는 팀을 위해, 더 효율적인 작업 방식과 팀 단위 운영 패턴을 8주에 걸쳐 정리합니다.

뉴스레터 로드맵

Week 1

Prompt Engineering에서 Agentic Engineering까지

개발 방식이 어떻게 AI와 함께 계획하고 검증하는 구조로 바뀌는지 설명합니다.

Week 2

Claude Code 작업 루프

바로 구현시키지 않고 조사, 계획, 구현, 테스트, 리뷰로 일을 맡기는 방법을 다룹니다.

Week 3

Context Engineering

CLAUDE.md와 프로젝트 메모리로 팀의 공통 맥락을 관리합니다.

Week 4

Skills

반복되는 리뷰, 커밋, 리팩토링, 장애 분석 작업을 재사용 가능한 skill로 표준화합니다.

Week 5

MCP

GitHub, Jira, 문서, 사내 API 같은 외부 시스템을 Claude Code와 연결하는 방식을 봅니다.

Week 6

SubAgent

큰 작업을 조사, 구현, 리뷰 역할로 나누어 context와 책임을 분리합니다.

Week 7

Hooks

테스트, 포맷팅, secret 검사, 위험 명령 차단 같은 팀 안전장치를 만듭니다.

Week 8

실전 명령어와 팀 플레이북

/simplify, /batch, /review 같은 팀용 slash command와 운영 가이드를 정리합니다.
