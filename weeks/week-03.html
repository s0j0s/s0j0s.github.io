Week 3 of 8 프로젝트 지침

Week 3

CLAUDE.md를 작업 기준으로 활용하기

약 3분 · 실무 활용 팁

저장소에 공유된 CLAUDE.md를 실제 작업에서 어떻게 적용하고, 부족한 내용은 어떻게 개선 후보로 남길지 다룹니다.

3줄 요약

  • CLAUDE.md는 Claude가 팀의 작업 규칙을 참고하게 하는 프로젝트 지침 파일입니다.
  • 작업 전에는 CLAUDE.md의 팀 규칙을 이번 작업 기준으로 정리하고, 작업 후에는 결과와 다시 비교합니다.
  • 오래된 명령이나 애매한 규칙을 발견하면 다음 작업을 위해 개선 후보로 남깁니다.

우리 팀은 이미 CLAUDE.md를 저장소에 공유해 사용하고 있습니다. 그래서 이번 글은 “어떻게 처음 만들까”보다 “작업할 때 어떻게 자연스럽게 적용할까”에 가깝습니다. 규칙이 많거나 서로 충돌하면 이번 작업에 필요한 기준이 흐려질 수 있습니다. 이럴 때는 적용할 규칙을 먼저 정리해볼 수 있습니다.

핵심은 간단합니다. 작업을 시작할 때 CLAUDE.md의 팀 규칙을 이번 작업 기준으로 정리하게 하고, 작업이 끝나면 결과가 그 기준과 맞는지 다시 보는 것입니다. 그리고 문서와 실제 코드가 어긋나는 부분을 발견하면 개선 후보로 남깁니다. 같은 문제를 다음 사람이 반복해서 만나지 않게 만드는 것이 목적입니다.

1. 작업 시작 전에 기준 확인하기

Claude Code는 작업을 시작할 때 프로젝트의 CLAUDE.md를 참고합니다. 다만 읽었다고 해서 모든 규칙이 항상 같은 중요도로 적용되는 것은 아닙니다. 그래서 중요한 작업을 맡길 때는 CLAUDE.md의 팀 규칙을 이번 작업에 어떻게 적용할지 먼저 정리해달라고 요청할 수 있습니다. 

바로 구현하지 말고 CLAUDE.md의 팀 규칙을 기준으로
이번 작업에 적용할 테스트 명령, 주의할 파일, 리뷰 포인트를 먼저 정리한 뒤
수정 계획을 제안해줘.

2. 수정 결과를 기준과 비교해보기

구현이 끝난 뒤에는 CLAUDE.md를 기준으로 결과를 다시 점검할 수 있습니다. 수정된 내용이 문서의 기준과 맞는지 확인해달라고 요청하면 리뷰 전에 놓친 부분을 한 번 더 점검할 수 있습니다. 

이번 수정 내용을 CLAUDE.md의 기준과 비교해서 다시 리뷰해줘.
어긴 규칙, 빠진 테스트, 위험한 변경 범위가 있으면 먼저 알려줘.

3. 문서 내용이 실제 작업과 다를 때

실제 작업을 하다 보면 CLAUDE.md에 없는 규칙, 오래된 테스트 명령, 애매한 표현을 발견할 수 있습니다. 개인만 알고 넘어가면 비슷한 혼동이 다시 생길 수 있습니다.

이번 작업에서만 처리한 경우

테스트 명령이 틀린 것 같은데
이번에는 내가 알아서 돌릴게.

개선 후보를 남긴 경우

CLAUDE.md의 테스트 명령이 현재 Gradle 설정과 맞지 않아.
이번 작업에서는 ./gradlew test를 사용했고,
CLAUDE.md에는 이 명령으로 수정하는 것을 제안해줘.

문서 수정 권한이 없어도 괜찮습니다. PR 설명이나 팀 채널에 “이번 작업에서 발견한 CLAUDE.md 개선 후보”로 남기는 것만으로도 다음 작업의 기준이 더 명확해집니다.

4. 개인 설정과 팀 지침 나누기

프로젝트의 CLAUDE.md는 팀원이 함께 사용하는 지침 파일입니다. 모든 프로젝트에 적용할 개인 설정은 ~/.claude/CLAUDE.md에, 현재 프로젝트에만 적용할 설정은 CLAUDE.local.md에 관리하고, 팀 문서에는 빌드, 테스트, 설계, 리뷰처럼 함께 참고할 내용을 남길 수 있습니다.

5. 길어질수록 가끔 정리하기

CLAUDE.md의 내용이 너무 많아지면 필요한 지침을 일관되게 따르기 어려울 수 있습니다. 오래된 규칙, 특정 사건의 회고, 예외가 많은 설명은 필요할 때 정리할 수 있습니다. 목적은 문서를 길게 만드는 것이 아니라 Claude가 필요한 내용을 찾기 쉽게 유지하는 것입니다.

6. Skill과 나누는 기준

여러 작업에서 계속 참고할 프로젝트 규칙은 CLAUDE.md에 두기 좋습니다. 반면 코드 리뷰나 릴리즈 노트 작성처럼 특정 작업에서만 꺼내 쓰는 절차와 참고 자료는 Skill로 분리하는 편이 적합합니다.

가볍게 참고해볼 점

Claude Code는 프로젝트의 CLAUDE.md를 불러오지만, 그 내용이 항상 같은 수준으로 반영된다고 보기는 어렵습니다. 중요한 작업을 맡길 때는 “읽었는지”보다 “이번 작업에 어떤 기준을 적용할지”를 먼저 정리하게 하는 편이 좋습니다. 

CLAUDE.md 기준으로 이번 작업에 적용할 규칙과 검증 방법을 먼저 정리해줘.
그 기준 안에서만 수정하고, 작업 후에는 어긋난 부분이 없는지 다시 확인해줘.

한 문장으로 정리

CLAUDE.md는 작업 전 기준 정리와 작업 후 점검에 활용하는 프로젝트 지침 파일입니다.
