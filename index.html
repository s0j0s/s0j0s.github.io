Claude Code Engineering Roadmap

8-week newsletter

Claude Code를 개인 도구에서 팀의 Engineering System으로

회사에서 이미 Claude Code를 쓰고 있는 팀을 위해, 더 효율적인 작업 방식과 팀 단위 운영 패턴을 8주에 걸쳐 정리합니다.

뉴스레터 로드맵

Week 1

Claude Code에게 일을 잘 맡기는 법

“고쳐줘”에서 한 걸음 나아가 조사, 작은 수정, 테스트 근거를 남기는 기본 습관을 다룹니다.

Week 2

바로 구현시키지 않는 Claude Code 작업 루프

조사, 계획, 승인, 구현, 테스트, diff 리뷰로 이어지는 기본 루프를 다룹니다.

Week 3

CLAUDE.md를 팀 작업 기준으로 활용하기

이미 배포된 CLAUDE.md를 작업 전후에 확인하고, 부족한 기준을 개선 후보로 남깁니다.

Week 4

Skills로 반복 작업 표준화하기

반복되는 리뷰, 커밋, 리팩토링, 장애 분석 작업을 재사용 가능한 skill로 표준화합니다.

Week 5

MCP로 외부 시스템 연결하기

GitHub, Jira, Confluence를 함께 읽게 해서 코드 밖 작업 맥락을 모으는 방식을 봅니다.

Week 6

SubAgent로 큰 작업 나누기

큰 작업을 조사, 구현, 리뷰 역할로 나누어 context와 책임을 분리합니다.

Week 7

Hooks로 안전장치 만들기

JUnit 테스트, 정적 검사, secret 검사, 위험 명령 경고 같은 팀 안전장치를 만듭니다.

Week 8

팀 플레이북: Claude Code를 팀의 개발 시스템으로 만들기

팀용 slash command와 운영 기준을 정리해 개인 팁을 팀의 개발 시스템으로 바꿉니다.
