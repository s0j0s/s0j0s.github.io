Claude Code Engineering Roadmap

8-week newsletter

팀에서 조금 더 편하게 쓰는 Claude Code

회사에서 이미 Claude Code를 쓰고 있는 팀을 위해, 각자 써본 팁과 필요할 때 꺼내볼 만한 작업 패턴을 8주에 걸쳐 가볍게 정리합니다.

뉴스레터 로드맵

Week 1

Claude Code에게 일을 잘 맡기는 법

작업을 맡기기 전에 붙여볼 요청 문장과, 조사·테스트 근거를 남기는 습관을 살펴봅니다.

Week 2

바로 구현하기 전에 살펴보는 작업 루프

조사, 계획, 방향 확인, 구현, 테스트, diff 리뷰로 이어지는 흐름을 가볍게 써봅니다.

Week 3

CLAUDE.md를 작업 메모처럼 활용하기

이미 배포된 CLAUDE.md를 참고하고, 실제 작업에서 발견한 개선 후보를 가볍게 남겨봅니다.

Week 4

Skills로 반복 설명 줄이기

자주 반복한 요청 하나를 Skill 초안으로 남기고 작은 PR에 시험해봅니다.

Week 5

MCP로 외부 시스템 연결하기

GitHub, Jira, Confluence를 어떤 순서로 읽으면 편한지, 쓰기 작업은 어디까지 맡길지 살펴봅니다.

Week 6

SubAgent로 큰 작업 나누기

큰 작업에서 조사·구현·리뷰 역할을 나눠보면 편한 경우를 살펴봅니다.

Week 7

Hooks로 잊기 쉬운 확인 알려주기

처음에는 자동 실행보다 테스트 확인 리마인더처럼 부담 없는 Hook부터 써봅니다.

Week 8

팀에서 편하게 꺼내 쓰는 Claude Code 메모

8주 동안 써본 요청, Skill, MCP, SubAgent, Hook을 작은 팀 메모로 묶어봅니다.
